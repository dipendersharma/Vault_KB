What are Configuration Templates?

AutoRABIT utilizes configuration templates to provision parameters to define the data relationship between CPQ objects. You can use configuration templates to deploy the selected objects that are part of the template to your sandboxes or to a specific environment. Those objects will retrieve dependent and intermediate objects by default. 

For ex- a single object record can be referred to multiple objects records as required and this defines the data relationship, so the selection of these objects is a configuration template for your application. 

Create a Configuration Template

This article deals with the step-by-step procedure to create a new template.

  1. In your browser, navigate to: https://{my-organization}.autorabit.com
  2. Enter your login credentials.
  3. Click Login.
  4. Click on the Salesforce CPQ tile, which will navigate to the App Configuration Migrator portal.
  5. Click on the Configuration Templates box.
  6. Select your source Salesforce Org. 
  7. Click Fetch Objects to retrieve all the objects that are available in the selected source org.
  8. Select the item you want to create a template from on the following screen. For ex- Account Brand, Relationship, Checklist, etc.,
  9. You can use the Search filter to search through your objects and the Menu button to quickly filter your Standard/Custom objects.
  10. To view your chosen items, you can slide the Show Selected button to the right side.
  11. Based on the relationship between the objects, the AutoRABIT application will segregate the object sets, and it is mandatory to apply a filter on each bucket. By specifying criteria in the Filters section, you can extract records within a specified limit.
  12. Apply the filter(s) using the steps as shown below:
    1. Identify an object in each bucket to which you would like to add a filter. Click on the Filter icon.
    2. You can use either Add Group or Add Condition on your selected object to build a SOQL query in both the condition (AND / OR). To add a condition to your selected objects, for example, click the Add Condition button. To add multiple conditions, click Add Condition and then repeat the process.
    3. You can even filter the details via date or a date literal. A date literal is a fixed expression that represents a relative range of time, such as last month, this week, or next year. (Refer HERE for the list of data literals supported)
    4. The filter query gets added and gets displayed in the filter box. Nevertheless, multiple queries can be added based on your requirement. Validate your query to see if the query entered is stable or not. Additionally, you'll be able to view a number of records that will get fetched.
    5. The Limit box will limit the number of records to be extracted from the source, giving a value in this field.
  13. Similar steps are to be followed for objects in other buckets as well. Thereafter, click on Next.
  14. On the next step of the wizard, youâ€™ll find the summary page for the configuration template that you're about to create. You can view the details and the filters that you have applied for the last time before the template gets created. On this page, you will need to:
    • Give the template a name 
    • Assign the template a version number
    • Give a short description of your template in the Template Notes field (optional).
  15. Click Save and you will be redirected to the App Configuration Templates summary page where you can track your template recently created.

App Configuration Templates Summary

Configuration Template Summary screen is the place where you can see all your created templates to date. For each template created, you have several options to perform here such as viewing the changelog, version logs, viewing the data templates, or deleting the unwanted templates from your sandbox.

To manage the template, simply click on the three dots in the template's corner. It will display all of the options listed below.

Change Log

View the changelog information that you have added during the template creation at the Template Notes field.

Version Log

This lists down the template versions that the template belongs to.

View Data Template

You can see a list of the objects and data that were retrieved as part of the template.

Delete a Template

Deletes the no longer required template. This process cannot be undone.
